## Context

当前管理后台前端使用 mock token，后端未启用 JWT 鉴权；会话查询只能按用户返回全量列表；模板管理没有实际接口。第二阶段要在保持现有三端可运行的前提下补齐可运营能力。

## Goals / Non-Goals

**Goals:**
- 提供 JWT 登录接口并保护管理端核心 API。
- 提供会话分页查询能力（page/pageSize/total/items）。
- 提供模板管理 CRUD 接口并接入后台页面。

**Non-Goals:**
- 不实现细粒度 RBAC 权限模型。
- 不引入数据库迁移，仅使用内存存储占位。
- 不改造小程序侧鉴权流程。

## Decisions

- 使用简化 JWT（对称密钥）与固定管理员账号进行首版闭环验证。
- 将会话分页结果统一为 `PagedResult<T>`，后续可复用于内容与订单。
- 模板管理使用单独控制器 `/api/v1/templates`，避免与会话领域耦合。

## Risks / Trade-offs

- [风险] 固定管理员账号不适合生产 → [缓解] 明确仅用于开发阶段，后续替换为真实用户表。
- [风险] 内存存储重启丢数据 → [缓解] 当前阶段仅用于接口联调，后续接入持久化。
- [风险] 前端 token 持久化逻辑不当可能导致状态错乱 → [缓解] store 初始化时显式从 localStorage 恢复。
