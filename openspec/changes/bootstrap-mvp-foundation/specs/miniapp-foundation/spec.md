## ADDED Requirements

### Requirement: 小程序必须提供MVP基础页面与导航
系统 MUST 提供首页、占卜输入页、占卜结果页、历史记录页、个人中心页，并可在小程序中完成页面跳转。

#### Scenario: 用户进入小程序首页
- **WHEN** 用户首次打开小程序
- **THEN** 系统展示首页并提供进入占卜流程与资讯入口

#### Scenario: 用户查看历史记录
- **WHEN** 用户在个人中心点击历史入口
- **THEN** 系统跳转历史记录页并请求历史会话列表接口

### Requirement: 小程序必须通过统一API层访问后端
系统 MUST 在小程序端提供统一请求封装，支持鉴权头、错误码处理与基础重试策略。

#### Scenario: 请求成功返回占卜结果
- **WHEN** 小程序调用创建占卜会话接口返回成功
- **THEN** 系统将结果渲染到结果页并缓存最近一次会话摘要

#### Scenario: 请求失败提示用户
- **WHEN** 后端返回业务错误码
- **THEN** 系统展示可读错误提示并记录调试日志
