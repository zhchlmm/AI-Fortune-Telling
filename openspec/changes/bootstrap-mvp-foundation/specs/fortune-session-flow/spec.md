## ADDED Requirements

### Requirement: 占卜会话流程必须支持多类型占卜输入
系统 MUST 支持至少两类占卜输入（塔罗、星座），并统一映射为会话请求模型。

#### Scenario: 塔罗占卜请求
- **WHEN** 用户选择塔罗并提交问题与抽牌参数
- **THEN** 系统创建会话并返回塔罗解读结果

#### Scenario: 星座占卜请求
- **WHEN** 用户选择星座并提交星座与时间范围参数
- **THEN** 系统创建会话并返回星座运势结果

### Requirement: 占卜结果必须可追踪与可回放
系统 MUST 在会话记录中持久化输入参数摘要、模型版本标识与结果摘要。

#### Scenario: 后台查看会话详情
- **WHEN** 管理员在会话查询页打开任一会话
- **THEN** 系统展示该会话的输入摘要、结果摘要与创建时间
